## State Management Summary

### Key Stores and Their Responsibilities

1. **authStore**
   - User authentication state
   - JWT token management
   - User profile data

2. **logbookStore**
   - Practice entries (Map structure)
   - Local/cloud sync management
   - Filtering and search

3. **repertoireStore**
   - Repertoire items
   - Goals and progress
   - Score metadata cache

4. **scoreStore**
   - Available scores
   - User library
   - Collections

5. **practiceStore**
   - Active practice sessions
   - Timer management
   - Auto-logging preferences

6. **reportingStore**
   - Report filters
   - View preferences
   - Cached analytics

### Data Persistence

- **Authenticated users**: Cloud sync with local cache
- **Anonymous users**: Local storage only
- **Offline mode**: Queue changes for sync
- **Conflict resolution**: Last-write wins with timestamps

### Navigation Patterns

- **Protected routes**: Redirect to landing if not authenticated
- **Deep linking**: Maintain context through authentication
- **Tab persistence**: Remember last active tab
- **Back navigation**: Preserve filters and scroll position
